# Tampopo Holding Site Design

## Summary

Build a basic static holding site for タンポポ合同会社 / Tampopo LLC. The site will be Japanese-first and bilingual, suitable for free GitHub Pages hosting, and intentionally small because product-specific sites live elsewhere.

## Goals

- Present the organization clearly as タンポポ合同会社, with Tampopo LLC as the English name.
- Link visitors to the current build product at `https://kanna.build`.
- Provide a contact path for `admin@tampopomyoko.com` without placing the full email address directly in static HTML.
- Keep hosting free and simple through GitHub Pages.

## Non-Goals

- No CMS, backend, analytics, newsletter, blog, or multi-page product directory.
- No contact form provider in the first version.
- No language switcher; Japanese content leads, English appears as secondary supporting copy.

## Site Structure

The site will be a single static page:

- Hero/header: company name in Japanese, English name nearby.
- Short purpose statement: a concise bilingual note that this is the company holding site and product information is available on product sites.
- Product section: a prominent link to Kanna at `https://kanna.build`.
- Contact section: a contact button/link generated by JavaScript.
- Footer: company name and current year.

## Contact Behavior

The contact action will use light JavaScript obfuscation:

- Store email parts separately, for example local part and domain fragments.
- Assemble the `mailto:` URL only when the page loads or when the user clicks the contact control.
- Display contact text without embedding the complete email address in the HTML source.

This reduces casual scraping, but it is not a security boundary because browser-delivered code can be inspected.

## Implementation Architecture

Use a minimal static site compatible with GitHub Pages:

- `index.html` for markup and copy.
- `styles.css` for responsive layout and visual polish.
- `script.js` for contact email assembly.
- Optional `README.md` with GitHub Pages deployment notes.

The site should work when opened directly from the filesystem and when served through GitHub Pages.

## Visual Direction

Use a restrained corporate profile style rather than a marketing landing page:

- Clean Japanese-first typography and compact sections.
- Calm, professional layout with enough whitespace to feel intentional.
- Product link should be obvious but not presented as the whole company site.
- Avoid decorative image dependencies so the site remains lightweight and durable.

## Testing

Manual verification is sufficient for this scope:

- Open `index.html` locally.
- Confirm responsive layout at desktop and mobile widths.
- Confirm the Kanna link opens `https://kanna.build`.
- Confirm the contact control creates a `mailto:` for `admin@tampopomyoko.com`.
- Confirm no full email address appears in `index.html`.
